~java (source code) 원시코드 ->>     <<컴파일링 (0,1)>>     ~.class(object code) 목적코드
 알파벳 + 숫자 + 기호                                       0과 1로 구성되어있다. = 기계어
 (사람이 알아볼수 있는)                                      (machine language)
                                                           (bin 폴더에 있음) 자바가상머신 끌어와서 그안에서 OS랑 교신해서 실행됨
                                                            로더가 안붙어있다.
                                                            수정한 부분만 컴파일한다 (빌드)
                                                            자바가상머신한테 위임한다.(맡긴다.) 메모리의 상대적인 주소를 잡아준다.
                                                            물리적인 주소를 잡아주는건 OS다.

                                                            클래스가 만들어질때마다 파일이 생성된다. (bin 폴더에 있다.)




~.c(C언어), ~.cpp(C++)  ->>      << compile >>      ~.obj               ->> <<link>> ~.exe (execute) 파일 배포
                            
                             (OS에 내장되어있음 - 가상머신따로 없음)           링크과정에서 로더를 ~.obj 덮붙힌다.
